# Big Data Midterm Project

## Overview
This midterm project requires you to implement a Big Data data-processing / analysis workflow using **Google Colab**, **Jupyter Notebook**, or a **Python script**. You must upload your work to a **GitHub repository** and submit required proof files in **Canvas** (including Zoom verification screenshots).

---

## What to Upload to GitHub (Required)
Your GitHub repository must include:

- **Notebook or code file**
  - `*.ipynb` (Google Colab / Jupyter Notebook) **OR**
  - `*.py` (Python script)
-  **README.md** (this file)
-  Dataset file(s) (only if allowed/required for your class)
-  Output files (optional): screenshots, plots, result CSVs, etc.

### Recommended Repo Structure
```text
BigData-Midterm-Project/
├── README.md
├── midterm_project.ipynb        # OR midterm_project.py
├── data/                        # optional
│   └── dataset.csv
└── outputs/                     # optional
    ├── results.png
    └── summary.csv
```

---

## What to Upload to Canvas (Required)
Upload **ALL** of the following to Canvas:

1.  **Project file** (`.ipynb` or `.py`) (same file you uploaded to GitHub)
2.  **Screenshot of your GitHub repository** showing the uploaded file(s)
3.  **Screenshot of the Zoom call** with your work clearly visible (required for verification)

---

## Zoom Verification (Required)
During the Zoom call:

- All group members must be present (if it is a group submission).
- You must **show your running code / notebook output** on screen.
- Take a **clear screenshot** that shows:
  - Zoom meeting window (participants visible if possible)
  - Your notebook/code and results visible on screen

---

## Execution Requirements
Your notebook/script should include:

- Problem statement / objective
- Dataset description
- Data preprocessing steps
- Implementation (analysis/model/queries)
- Results (metrics/graphs/output)
- Conclusion

Make sure your code runs without errors and produces outputs.
